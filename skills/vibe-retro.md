# /vibe:retro — Retrospective & Learning Capture

## Purpose
Analyze what went well, what could improve, and capture lessons learned.

## Input
- `.vibe/state.json` (completed phases)
- `.vibe/harness-report.json`
- `.vibe/review-report.json`
- Telemetry data

## Process
1. Analyze phase durations and outcomes
2. Identify bottlenecks and blockers
3. Calculate success metrics
4. Generate retro feedback
5. Store learnings in OKF bundle

## Output
- `.vibe/retro-notes.md` — Retro analysis
- Updated OKF bundle (new learnings)
- Updated `.vibe/evolution.json`

## Quality Gates
- [ ] All phases reviewed
- [ ] Lessons captured in OKF
- [ ] Action items identified
- [ ] Metrics recorded
