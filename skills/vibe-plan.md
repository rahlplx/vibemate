# /vibe:plan — Multi-Perspective Review

## Purpose
Review the design document from multiple perspectives (CEO, Engineering, Design) and create a task plan.

## Input
- `.vibe/design-doc.md`
- OKF bundle decisions

## Process
1. **CEO perspective**: Business value, ROI, market fit
2. **Engineering perspective**: Technical feasibility, complexity, risks
3. **Design perspective**: User experience, accessibility, consistency
4. Synthesize feedback into actionable tasks
5. Create milestone-based task plan

## Output
`.vibe/task-plan.md` containing:
- Milestones with deliverables
- Task breakdown per milestone
- Dependencies between tasks
- Complexity estimates
- Risk assessment

## Quality Gates
- [ ] All perspectives addressed
- [ ] Tasks are atomic (one concern each)
- [ ] Dependencies identified
- [ ] Complexity scores assigned
