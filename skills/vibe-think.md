# /vibe:think — Product Strategy & Design Thinking

## Purpose
Analyze the project description, identify user needs, define success metrics, and create a design document.

## Input
- Project description (from user or handoff)
- Existing OKF bundle (if any)

## Process
1. Parse the project description for key requirements
2. Identify user personas and their needs
3. Define success metrics (measurable)
4. Set scope boundaries (in/out)
5. Identify technical constraints
6. Generate design document

## Output
`.vibe/design-doc.md` containing:
- Project summary
- User personas
- Core requirements (prioritized)
- Success metrics
- Scope boundaries
- Technical constraints
- Architecture decisions (reference OKF bundle)

## Quality Gates
- [ ] All requirements have acceptance criteria
- [ ] Success metrics are measurable
- [ ] Scope is clearly bounded
- [ ] Technical constraints documented
